# tree-sitter-glsl-spec

See @README.md for project overview, usage, and development commands.

## Project layout

```
grammar.js              Main grammar (source of truth)
keywords.js             Keyword tables (types, qualifiers, storage classes)
extensions.js           Khronos extension registry (extra rules, keywords, builtins)
builtin.js              Built-in function/variable/constant tables
queries/
  highlights.in         Template - edit this, do NOT edit highlights.scm
  highlights.scm        Generated by: node scripts/generate_highlights.js
  *.scm                 Other queries are hand-written
test/corpus/            Tree-sitter test corpus
src/                    Generated C parser - never edit
specification/          Git submodule (KhronosGroup/GLSL) for spec-scraping scripts
```

## Grammar conventions

- Every BNF production from the GLSL 4.60 spec has a corresponding rule, documented with the original grammar notation in comments.
- Rules prefixed with `_` are hidden (not in the AST).
- The `OPT` object at the top of `grammar.js` controls parser features: `MACRO_PARSING`, `MACRO_EXPANSION`, `MULTILINGUAL`, `ESSL`.
- Extension grammar rules live in `extensions.js`, pulled in via `collectExtensionRules()` / `extRule()`.

## Development conventions

If you're not Claude:
- See `.claude/rules/*.md` files for file-specific rules that should be applied automatically.
- See `.claude/skills/**/SKILL.md` for development workflow preferences.
- Run `.claude/skills/update-spec/scripts/check.sh` to check if grammar needs to be updated to match the specification.
